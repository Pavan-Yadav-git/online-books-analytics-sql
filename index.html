📖 Project Documentation & Problem Statement

Analyzing the Online Bookstore relational database model.

1. Business Problem Scenario

An emerging **Online Bookstore** has compiled data across three separate operational sectors: inventory items (**Books**), client demographic info (**Customers**), and transaction records (**Orders**).

Management requires data-driven answers to answer fundamental operational questions, identify trends, and automate replenishment strategies.

2. Schema Description

The database schema consists of three normalized entities linked via primary and foreign key relationships:

  • Books: Manages the catalog. Tracks identifiers, titles, author credentials, literary genre classifications, publication year, store price, and warehouse stock units.
  • Customers: Logs membership registry. Contains contact coordinates, emails, telephone digits, and regional addresses (city, country).
  • Orders: Chronicles commercial activity. Captures sales dates, buyer IDs, product IDs, physical quantities purchased, and cash total amounts.

3. Technical Architecture

This dashboard showcases the queries in a fully serverless environment:

  • WebAssembly Core: SQLite compile compiled to WASM (`sql.js`) loads parsed records in client-side memory.
  • No Latency Mocking: Instead of fetching API mock files, queries are directly evaluated on the WebAssembly runtime in milliseconds.
  • Responsive UI: Built using glassmorphic UI elements and CSS layout tokens.
